How x402 Works
01
Request
Agent hits endpoint with no payment header
→
02
402 Response
Server returns payment details: asset, amount, payTo address
→
03
On-chain Tx
Agent sends USDC on Base, gets txHash
→
04
Re-request
Agent retries with X-PAYMENT: <txHash+sig>
→
05
Verify + Serve
Server verifies on-chain, returns data. ~200ms total.

Architecture & Extending

Current Stack

  • Express.js — HTTP server, port 4020
  • x402 Middleware — src/middleware/x402.js
  • Mock Verifier — src/services/mockVerifier.js
  • CoinGecko API — live price data, 60s cache
  • CORS + Morgan — logging & cross-origin

To Make It Real (Production)

  • Replace mockVerifier.js with ethers.js Base RPC call
  • Verify USDC tx to your payTo address on Base
  • Add Redis/DB for replay attack protection (store used txHashes)
  • Wire up Moralis/Alchemy for real whale tracker data
  • Call real protocol APIs for funding rates (Hyperliquid, dYdX)
  • Register ERC-8004 agent identity on Base for discoverability
  • List on Apiosk for instant AI agent discovery
